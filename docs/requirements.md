# MP3 黑胶播放器需求文档

> 版本：v1.0
> 日期：2026-07-03
> 基于 `App.js`（feature/song-list 分支）现有功能整理

---

## 1. 项目概述

### 1.1 产品定位
一款仿网易云音乐风格的黑胶唱片 MP3 播放器移动应用，以旋转黑胶唱片为核心视觉元素，提供沉浸式音乐播放体验。

### 1.2 目标用户
- 喜爱复古/怀旧音乐播放界面的用户
- 希望在移动端获得黑胶唱片视觉体验的听众

### 1.3 技术栈
| 项目 | 版本 |
| --- | --- |
| React Native | 0.75.4 |
| React | 18.3.1 |
| react-native-track-player | 4.1.1 |
| @react-native-community/slider | 5.2.0 |
| 平台 | iOS（主）/ Android |

## 2. 功能模块

应用包含两个核心界面，通过应用内状态切换：

```
┌──────────────┐     选择歌曲     ┌──────────────┐
│  歌单列表页   │ ─────────────────▶ │  黑胶播放页   │
│ PlaylistScreen│ ◀───────────────── │ PlayerScreen │
└──────────────┘     返回按钮      └──────────────┘
```

### 2.1 歌单列表页（PlaylistScreen）

#### 2.1.1 功能描述
展示所有可播放曲目，用户点击任意曲目即进入播放页并开始播放该曲目。

#### 2.1.2 界面要素

| 要素 | 说明 |
| --- | --- |
| 顶部红色光晕 | 背景顶部 `#C20C0C` 5% 透明度的渐变光晕，高度 280 |
| 列表头部 | 标题"播放列表"（22px 白色加粗）+ 副标题"共 N 首"（13px 灰色） |
| 列表项 | 左侧 48×48 圆角缩略图；中部歌曲名 + 艺术家；右侧当前播放标记 |
| 当前播放标记 | 正在播放的曲目：标题变红 `#C20C0C`，右侧显示红色 ▶ 图标 |
| 分隔线 | 细线，左侧缩进 84（与缩略图+文字对齐） |
| 背景 | 深色 `#1a1a1a` |

#### 2.1.3 交互行为
- 点击列表项：切换到播放页，开始播放对应曲目
- 当前播放曲目高亮显示

---

### 2.2 黑胶播放页（PlayerScreen）

#### 2.2.1 功能描述
以旋转黑胶唱片为主视觉，展示当前曲目信息、歌词、进度，并提供播放控制。

#### 2.2.2 界面要素

**顶部栏（topBar）**
- 左侧：返回按钮 `‹`（点击返回歌单列表）
- 中间：当前歌曲名（15px 白色，单行省略）
- 右侧：分享图标 `⤴`

**黑胶舞台（vinylStage）**
- **黑胶唱片**（340px 圆形，深黑 `#0a0a0a`，带阴影）
  - 3 道唱片纹路（groove rings）
  - 中心圆形专辑封面（220px，白色边框）
  - 中心红色小圆点标签（16px，`#C20C0C`）
- **唱针**（tonearm，130×130，定位在唱片右上）
  - 播放时：落下至 20°
  - 暂停时：抬起至 -30°
  - 切换动画：400ms 缓动

**歌词区（Lyrics）**
- 高度 90，显示 3 行（上一行 / 当前行 / 下一行）
- 当前行：17px 白色加粗
- 相邻行：14px 灰色 `#b3b3b3`，透明度 0.4
- 当前行索引由 `position/duration * lyrics.length` 推算

**进度条（Slider）**
- 红色已播放轨道 `#C20C0C`
- 白色滑块
- 灰色未播放轨道 `#ffffff20`
- 支持拖动跳转（onSlidingComplete）

**时间显示**
- 左侧：当前播放位置
- 右侧：总时长
- 格式：`M:SS`（使用 tabular-nums 等宽数字）

**控制栏（Controls）**
- 从左到右：
  1. 循环模式切换 `🔁`（开启时红色，关闭时灰色）
  2. 上一曲 `⏮`
  3. 播放/暂停 `▶` / `⏸`（64×64 白色描边圆形按钮）
  4. 下一曲 `⏭`
  5. 列表图标 `☰`

#### 2.2.3 交互行为

| 操作 | 行为 |
| --- | --- |
| 点击返回 `‹` | 返回歌单列表页 |
| 点击播放/暂停 | 切换播放状态；唱针落下/抬起；唱片开始/停止旋转 |
| 点击上一曲 | 切换到队列中的上一首，更新封面/标题/歌词 |
| 点击下一曲 | 切换到队列中的下一首，更新封面/标题/歌词 |
| 点击循环切换 | 切换循环开关（仅 UI 状态变化） |
| 拖动进度条 | 跳转到指定位置 |
| 播放中 | 唱片持续旋转（20 秒/圈，线性匀速） |
| 暂停 | 唱片停止旋转，保留当前角度；恢复播放时从该角度继续 |

---

### 2.3 播放服务（service.js）

通过 `react-native-track-player` 注册远程控制事件，支持系统媒体控制：

| 事件 | 行为 |
| --- | --- |
| `remote-play` | 调用 `TrackPlayer.play()` |
| `remote-pause` | 调用 `TrackPlayer.pause()` |
| `remote-stop` | 调用 `TrackPlayer.stop()` |
| `remote-next` | 调用 `TrackPlayer.skipToNext()` |
| `remote-previous` | 调用 `TrackPlayer.skipToPrevious()` |

## 3. 数据模型

### 3.1 曲目结构（Track）

```typescript
interface Track {
  id: string;          // 唯一标识
  url: string;         // 音频文件 URL
  title: string;       // 歌曲名
  artist: string;      // 艺术家
  artwork: string;     // 封面图 URL（600×600）
  lyrics: string[];    // 歌词数组（按行）
}
```

### 3.2 内置歌单

应用启动时加载 3 首示例曲目（来自 SoundHelix 公开示例音频），封面使用 picsum.photos 占位图，歌词为 8 行占位中文文本。

## 5. 已知限制与待办

### 5.1 当前限制

1. **循环模式未生效**：循环按钮仅切换 UI 状态，未实际配置 `TrackPlayer` 的 `RepeatMode`，单曲循环功能需补充。
2. **列表图标无功能**：控制栏右侧 `☰` 图标未绑定 `onPress`，目前为视觉占位（实际返回列表通过顶部 `‹` 按钮）。
3. **歌词非时间同步**：歌词为占位文本，按 `position/duration` 比例推算行索引，非真实 LRC 时间轴同步。
4. **示例曲目有限**：仅内置 3 首 SoundHelix 示例音频，无在线搜索/导入功能。
5. **无错误处理**：网络音频加载失败、播放器初始化失败等场景未做兜底提示。
6. **无自动化测试**：项目未配置 UI/单元测试，验证依赖 iOS 模拟器人工检查。
7. **仅单文件实现**：所有组件集中于 `App.js`，未做模块拆分。

### 5.2 后续可拓展方向

- 接入真实歌词解析（LRC 格式）
- 实现单曲/列表循环切换
- 增加收藏、最近播放、搜索
- 支持本地音频文件导入
- 播放队列管理
- 模块化代码拆分（`src/screens/`、`src/components/`）
- 增加深色/浅色主题切换
- Android 平台适配验证
